# スマートグラスアプリケーション 要件定義書

## 1. 基本情報

| 項目 | 内容 |
| --- | --- |
| プロジェクト名 | スマートグラス向けAI/ARアプリケーション開発 |
| 開発者 | 小松琢磨、中村壮馬 |
| 対象デバイス | INMO Air3 |
| 想定用途 | 人物記憶・視界ログの記録・検索、人物HUD表示、ARによる技能習得支援、遠隔作業支援、スマートグラスSDK開発 |
| 本書の目的 | 開発対象アプリケーションの目的、機能、検証項目、調達要件を明確化する |

## 2. 背景

スマートグラスは、ユーザーの視界、音声、空間位置、表示領域を同時に扱えるデバイスである。従来のスマートフォンやPCでは実現しにくい、一人称視点に密着した記憶支援、技能習得支援、遠隔作業支援を実現できる可能性がある。

本プロジェクトでは、INMO Air3 のカメラ、6DoF空間認識、ディスプレイ、音声入出力、ストリーミング機能を活用し、スマートグラスならではの主要アプリケーションを開発・検証する。

また、アプリケーションごとにデバイス制御処理を個別実装すると開発効率と保守性が低下するため、カメラ、表示、音声、センサー、通信、AI連携を共通化するスマートグラスSDKも開発対象とする。

## 3. 開発目的

本プロジェクトの目的は、スマートグラス上で実用性の高いAI/ARアプリケーションを開発し、以下を検証することである。

- 人物を中心としたAIセカンドブレインの実現可能性
- カメラに映った人物へ記憶情報を紐付け、視界上に提示する体験の実現可能性
- 空間認識とAR表示を用いた直感的なスキル習得支援
- POVストリーミングとARマーカーを用いた遠隔作業支援
- 実機上での表示品質、処理性能、通信遅延、操作性
- LLMおよび画像認識処理に伴うコストと処理負荷の最適化
- スマートグラス向けアプリを継続的に開発するための共通SDK基盤の整備

## 4. 対象アプリケーション

### 4.0 スマートグラスSDK

#### 概要

INMO Air3上で動作するアプリケーション開発を効率化するため、カメラ、ディスプレイ、音声、センサー、通信、AI連携などのデバイス機能を抽象化した共通SDKを開発する。

#### 想定利用シーン

- AIセカンドブレインから定期的にカメラ画像を取得する
- ARアプリから画面上にフローティングUIやマーカーを描画する
- 遠隔支援アプリからPOV映像を配信する
- 音声入力、音声出力、タッチ操作、リング操作を統一的に扱う
- 実機SDK/APIの差分を吸収し、アプリ側の実装を簡潔にする

#### 主要機能

- カメラ画像・動画取得API
- ディスプレイ描画・フローティングUI表示API
- 音声入力・音声出力API
- IMU、姿勢、空間認識情報の取得API
- タッチパッド、リング、ボタン操作の入力API
- POVストリーミングAPI
- AI認識・タグ付け処理との連携API
- 権限管理、ログ、エラー処理、設定管理
- 実機がない環境でのモック・シミュレーターAPI

### 4.1 AIセカンドブレイン

#### 概要

ユーザーの視界に入った人物を中心に、顔、名前、関係性、会った日時、会話メモ、場所などを記憶し、再会時や検索時に必要な情報を提示する拡張記憶アプリケーション。

初期段階では、人物の自動識別を完全自動化せず、ユーザーによる音声登録や確認を組み合わせる。誤認識時のリスクを下げるため、信頼度が低い場合は断定せず候補として表示する。

#### 想定利用シーン

- 目の前にいる人の名前や前回会った日時を確認する
- 昨日名刺交換した人の名前を確認する
- 複数人が映ったとき、それぞれの人物に紐付くメモを確認する
- 会話前に、その人との関係性や過去の会話メモを確認する
- 最後に車の鍵を置いた場所を確認する
- 最近訪問した場所や見た物を検索する
- 視界ログから特定の人物、物体、シーンを探す

#### 主要機能

- 視界画像の手動または定期キャプチャ
- 画像内の人物検出
- 顔画像または人物特徴の登録
- 人物プロフィール、会話メモ、会った日時の管理
- 人物候補の検索・照合
- カメラに映った人物付近へのHUD表示
- 画像からの物体・文字・場所情報の抽出
- 音声入力による自然言語検索
- 視覚ログの時系列保存
- 検索結果のスマートグラス画面への表示
- プライバシーに配慮した保存・削除・除外設定

### 4.2 ゴースト・インストラクター

#### 概要

6DoF空間認識と高精細ディスプレイを活用し、現実の物体に半透明の3D映像を重畳表示する学習支援アプリケーション。

#### 想定利用シーン

- ピアノ鍵盤上に手本の指の動きを表示する
- 電子工作の部品配置やはんだ付け位置を表示する
- プラモデルや組み立て作業の手順をARで表示する
- 作業者の動きと手本の動きを比較する

#### 主要機能

- 現実物体の認識・トラッキング
- 3Dモデルまたは動画ベースの手本表示
- 半透明ARオーバーレイ表示
- 作業ステップの進行管理
- ユーザー動作との同期・差分表示
- 教材データの登録・切り替え

### 4.3 多腕アバター・ワークアシスト

#### 概要

ユーザーの一人称視点を遠隔地の支援者へ共有し、支援者が指示した位置や情報を作業者のAR視界内へリアルタイムに表示する遠隔作業支援アプリケーション。

#### 想定利用シーン

- ハードウェア組み立ての遠隔支援
- DIY作業中の熟練者による指示
- 修理・点検作業の手順案内
- 教育現場での作業補助

#### 主要機能

- POV映像のリアルタイム配信
- 遠隔支援者向けWeb/スマホ画面
- 支援者によるポインター・マーカー描画
- 作業者のAR視界へのマーカー重畳
- 音声通話または音声指示
- 通信遅延を抑えた双方向同期

## 5. 優先順位

| 優先度 | 対象 | 理由 |
| --- | --- | --- |
| 最高 | AIセカンドブレイン | 人物記憶とHUD表示が本プロジェクトの中心価値であり、最初に体験価値を検証する必要があるため |
| 高 | スマートグラスSDK | AIセカンドブレインの実装過程で必要になった共通機能を切り出し、後続アプリへ展開するため |
| 中 | 多腕アバター・ワークアシスト | POVストリーミングとAR表示の実用性検証に適しているため |
| 中 | ゴースト・インストラクター | 空間認識と3D表示の検証価値は高いが、教材・トラッキング精度への依存が大きいため |
| 低 | その他アプリ | 実機検証後に追加判断するため |

## 8. データ要件

### 8.1 保存対象データ

| データ | 内容 |
| --- | --- |
| 視界画像 | 定期キャプチャされた画像 |
| 顔画像・人物特徴 | 人物登録や照合に使用する顔画像、特徴量、信頼度 |
| 人物プロフィール | 名前、関係性、所属、メモ、表示設定 |
| 人物接触履歴 | 会った日時、場所、会話メモ、関連画像 |
| メタデータ | 撮影時刻、位置情報、デバイス状態、信頼度 |
| AIタグ | 物体、人物、文字、場所、行動、イベント |
| 検索履歴 | ユーザーの質問、回答、参照ログ |
| AR教材 | 3Dモデル、手順情報、ガイド表示データ |
| 遠隔支援ログ | セッション情報、マーカー位置、通話状態 |

### 8.2 データ管理方針

- 個人情報を含む可能性のあるデータは、保存期間と削除方法を明確にする。
- 顔画像や人物特徴量は特に機微な情報として扱い、保存、表示、削除の方針を明確にする。
- 人物識別結果は信頼度を持たせ、低信頼度の場合は断定表示しない。
- 他人の個人情報を扱うため、実証段階では利用範囲と保存対象を限定する。
- 画像そのものを常時LLMへ送信せず、必要に応じてエッジ処理や圧縮・要約を行う。
- 検索用には画像全体ではなく、タグ、埋め込み、要約テキストの活用を検討する。
- 実証段階では、保存対象と送信対象をユーザーが明示的に把握できるようにする。

## 10. MVP範囲

初期MVPでは、人物中心のAIセカンドブレインを最優先とし、必要なSDK機能は実装過程で最小限切り出す。

| 項目 | MVP内容 |
| --- | --- |
| 人物検出 | カメラ画像から人物を検出する |
| 人物登録 | 検出した人物に名前、関係性、メモを登録する |
| 人物記憶 | 登録済み人物の会った日時、場所、メモを保存する |
| 人物照合 | カメラに映った人物と登録済み人物を照合する |
| HUD表示 | 照合できた人物の名前やメモを画面上に表示する |
| 音声検索 | 「この人誰だっけ」「中村さんと最後に会ったのいつ」などを検索する |
| データ管理 | 人物情報、顔画像、視覚ログを一覧・削除できるようにする |
| SDK | カメラ取得、簡易表示、音声入力、設定、モック実装を必要範囲で切り出す |

MVPの成功条件は、ユーザーが登録済み人物と再会したときに、スマートグラス上でその人物の名前、関係性、過去メモを確認できることである。複数人が映った場合は、少なくとも人物ごとに別ラベルを表示できることを目標とする。

## 12. 前提・制約

- 実機仕様、SDK、開発環境はINMO Air3の提供内容に依存する。
- 画像認識、音声認識、LLM処理には外部APIまたはローカル推論環境を利用する可能性がある。
- 実運用では、第三者の顔、会話、個人情報を扱う可能性があるため、プライバシー設計を必須とする。
- 長時間の常時キャプチャは、バッテリー、発熱、ストレージ、APIコストに影響する。
- AR表示の精度は、実機の空間認識性能とSDKの制約を受ける。
