# スマートグラスアプリケーション 要件定義書

## 1. 基本情報

| 項目 | 内容 |
| --- | --- |
| プロジェクト名 | Ray-Ban Meta向けAIアプリケーション開発 |
| 開発者 | 小松琢磨、中村壮馬 |
| 対象デバイス | Ray-Ban Meta glasses |
| 想定用途 | Androidアプリ経由のカメラ・音声取得、自作AIへの送信、人物記憶・視界ログの記録・検索、音声応答、スマホ管理画面 |
| 本書の目的 | 開発対象アプリケーションの目的、機能、検証項目、調達要件を明確化する |

## 2. 背景

スマートグラスは、ユーザーの視界と音声を一人称視点で取得できるデバイスである。従来のスマートフォンやPCでは実現しにくい、装着者の視点に密着した記憶支援、状況理解、音声対話を実現できる可能性がある。

本プロジェクトでは、Ray-Ban Meta glassesのカメラ、マイク、オープンイヤースピーカーをAndroidアプリから扱い、取得した画像・音声を自作AIへ送信する構成を開発・検証する。対象デバイスにはディスプレイがないため、ユーザーへの提示はスマホ画面、音声応答、通知を基本とする。

Meta AIは利用できない前提とし、音声認識、会話理解、画像理解、人物記憶、検索応答は自作AIまたは外部AI APIを組み合わせて実装する。

## 3. 開発目的

本プロジェクトの目的は、Ray-Ban MetaとAndroidアプリを組み合わせて実用性の高いAIアプリケーションを開発し、以下を検証することである。

- 人物を中心としたAIセカンドブレインの実現可能性
- カメラに映った人物・物体・文字・場所を自作AIで認識し、記憶情報へ紐付ける体験の実現可能性
- 音声入力を自作AIへ送り、スマホ画面または音声で応答する体験の実現可能性
- Ray-Ban MetaからAndroidアプリへカメラ・音声を取得する実装可否
- 実機上での取得品質、処理性能、通信遅延、操作性
- LLMおよび画像認識処理に伴うコストと処理負荷の最適化
- Androidアプリから自作AIサーバーへ安全に送信・保存・削除する基盤の整備

## 4. 対象アプリケーション

### 4.0 Android連携アプリ

#### 概要

Ray-Ban Meta glassesと接続し、カメラ画像・動画フレーム・音声を取得して自作AIへ送信するAndroidアプリを開発する。初期MVPでは、このAndroidアプリを最優先の実装対象とする。

#### 想定利用シーン

- Ray-Ban Metaのカメラ画像をAndroidアプリで取得する
- Ray-Ban Metaまたはスマホのマイク音声をAndroidアプリで取得する
- 取得した画像・音声を自作AIサーバーへ送信する
- 自作AIの応答をスマホ画面、通知、音声でユーザーへ返す
- 実機がない間はモックカメラ・スマホカメラでAI連携を検証する

#### 主要機能

- Ray-Ban Metaとの接続・権限管理
- カメラ画像・動画フレーム取得
- 音声入力・音声出力
- 自作AIサーバーへの画像・音声送信
- AI応答のスマホ画面・通知・音声提示
- 送信ログ、エラー、再接続、設定管理
- 実機がない環境でのモック・シミュレーター

### 4.0A 将来の共通SDK

#### 概要

Android連携アプリの実装が安定した後、カメラ、音声、通信、AI連携、保存、権限管理を共通化するSDKを必要範囲で切り出す。

#### 主要機能

- カメラ画像・動画取得API
- 音声入力・音声出力API
- AI送信・応答取得API
- AI認識・タグ付け処理との連携API
- 権限管理、ログ、エラー処理、設定管理
- 実機がない環境でのモック・シミュレーターAPI

### 4.1 AIセカンドブレイン

#### 概要

ユーザーの視界に入った人物を中心に、顔、名前、関係性、会った日時、会話メモ、場所などを記憶し、再会時や検索時に必要な情報を提示する拡張記憶アプリケーション。

初期段階では、人物の自動識別を完全自動化せず、ユーザーによる音声登録や確認を組み合わせる。誤認識時のリスクを下げるため、信頼度が低い場合は断定せず候補として表示する。

#### 想定利用シーン

- 目の前にいる人の名前や前回会った日時を確認する
- 昨日名刺交換した人の名前を確認する
- 複数人が映ったとき、それぞれの人物に紐付くメモを確認する
- 会話前に、その人との関係性や過去の会話メモを確認する
- 最後に車の鍵を置いた場所を確認する
- 最近訪問した場所や見た物を検索する
- 視界ログから特定の人物、物体、シーンを探す

#### 主要機能

- 視界画像の手動または定期キャプチャ
- 画像内の人物検出
- 顔画像または人物特徴の登録
- 人物プロフィール、会話メモ、会った日時の管理
- 人物候補の検索・照合
- スマホ画面または音声による人物情報提示
- 画像からの物体・文字・場所情報の抽出
- 音声入力による自然言語検索
- 視覚ログの時系列保存
- 検索結果のスマホ画面、通知、音声での提示
- プライバシーに配慮した保存・削除・除外設定

### 4.2 ゴースト・インストラクター

#### 概要

カメラ映像と自作AIの状況理解を活用し、スマホ画面または音声で手順や注意点を提示する学習支援アプリケーション。Ray-Ban Metaにはディスプレイがないため、ARオーバーレイはMVP対象外とする。

#### 想定利用シーン

- ピアノ練習中に次の練習ポイントを音声で提示する
- 電子工作の部品配置やはんだ付け位置をスマホ画面で確認する
- プラモデルや組み立て作業の手順を音声で案内する
- 作業者の動きと手本の動きを比較する

#### 主要機能

- 現実物体の認識・トラッキング
- スマホ画面または音声による手順提示
- 作業ステップの進行管理
- ユーザー動作との同期・差分表示
- 教材データの登録・切り替え

### 4.3 多腕アバター・ワークアシスト

#### 概要

ユーザーの一人称視点を遠隔地の支援者または自作AIへ共有し、指示や解析結果をスマホ画面・音声で返す遠隔作業支援アプリケーション。

#### 想定利用シーン

- ハードウェア組み立ての遠隔支援
- DIY作業中の熟練者による指示
- 修理・点検作業の手順案内
- 教育現場での作業補助

#### 主要機能

- POV映像のリアルタイム配信
- 遠隔支援者向けWeb/スマホ画面
- 支援者によるポインター・マーカー描画
- 作業者のスマホ画面または音声への指示提示
- 音声通話または音声指示
- 通信遅延を抑えた双方向同期

## 5. 優先順位

| 優先度 | 対象 | 理由 |
| --- | --- | --- |
| 最高 | Android連携アプリ | Ray-Ban Metaのカメラ・音声を取得して自作AIへ送る最小基盤が全アプリの前提になるため |
| 高 | AIセカンドブレイン | 人物記憶と検索応答が本プロジェクトの中心価値であり、Android連携後に最初に体験価値を検証するため |
| 中 | 共通SDK | Android連携アプリの実装過程で必要になった共通機能を切り出し、後続アプリへ展開するため |
| 中 | 多腕アバター・ワークアシスト | POVストリーミングと音声支援の実用性検証に適しているため |
| 低 | ゴースト・インストラクター | ディスプレイなしではAR重畳ができないため、音声・スマホ画面による支援に範囲を縮小する |
| 低 | その他アプリ | 実機検証後に追加判断するため |

## 8. データ要件

### 8.1 保存対象データ

| データ | 内容 |
| --- | --- |
| 視界画像 | 保存ボタン押下時、またはユーザーが明示的に許可したタイミングでキャプチャされた画像 |
| 記憶イベント | 保存ボタン押下時に固定された画像、音声、時刻、AI解析結果 |
| 顔画像・人物特徴 | 人物登録や照合に使用する顔画像、特徴量、信頼度 |
| 人物プロフィール | 名前、関係性、所属、メモ、表示設定 |
| 人物接触履歴 | 会った日時、場所、会話メモ、関連画像 |
| メタデータ | 撮影時刻、位置情報、デバイス状態、信頼度 |
| AIタグ | 物体、人物、文字、場所、行動、イベント |
| 検索履歴 | ユーザーの質問、回答、参照ログ |
| 教材データ | 手順情報、参考画像、動画、ガイド説明データ |
| 遠隔支援ログ | セッション情報、マーカー位置、通話状態 |

### 8.2 データ管理方針

- 個人情報を含む可能性のあるデータは、保存期間と削除方法を明確にする。
- 顔画像や人物特徴量は特に機微な情報として扱い、保存、表示、削除の方針を明確にする。
- 人物識別結果は信頼度を持たせ、低信頼度の場合は断定表示しない。
- 他人の個人情報を扱うため、実証段階では利用範囲と保存対象を限定する。
- 画像そのものを常時LLMへ送信せず、必要に応じてエッジ処理や圧縮・要約を行う。
- 検索用には画像全体ではなく、タグ、埋め込み、要約テキストの活用を検討する。
- 実証段階では、保存対象と送信対象をユーザーが明示的に把握できるようにする。

## 10. MVP範囲

初期MVPでは、Ray-Ban MetaとAndroidアプリの接続、カメラ・音声取得、自作AIへの送信を最優先とする。人物中心のAIセカンドブレインは、その上に載せる最初のAIユースケースとする。

| 項目 | MVP内容 |
| --- | --- |
| Android連携 | Ray-Ban Metaと接続し、カメラ・音声の取得可否を確認する |
| ボタン保存 | Androidアプリのボタン押下時に、その時点の画像と音声を記憶イベントとして保存する |
| AI送信 | 取得した画像・音声を自作AIサーバーへ送信し、応答を受け取る |
| AI記憶 | 画像説明、音声文字起こし、要約、タグを保存し、後から検索できるようにする |
| 応答提示 | AI応答をスマホ画面、通知、音声で提示する |
| 人物検出 | カメラ画像から人物を検出する |
| 人物登録 | 検出した人物に名前、関係性、メモを登録する |
| 人物記憶 | 登録済み人物の会った日時、場所、メモを保存する |
| 人物照合 | カメラに映った人物と登録済み人物を照合する |
| 情報提示 | 照合できた人物の名前やメモをスマホ画面または音声で提示する |
| 音声検索 | 「この人誰だっけ」「中村さんと最後に会ったのいつ」などを検索する |
| データ管理 | 人物情報、顔画像、視覚ログを一覧・削除できるようにする |
| 共通モジュール | カメラ取得、音声入力、AI送信、設定、モック実装を必要範囲で切り出す |

MVPの成功条件は、ユーザーがRay-Ban Metaで取得したカメラ画像と音声をAndroidアプリ経由で自作AIへ送り、AIの応答をスマホ画面または音声で確認できることである。人物記憶の成功条件は、登録済み人物と再会したときに、その人物の名前、関係性、過去メモを確認できることである。

## 12. 前提・制約

- 実機仕様、SDK、開発環境はRay-Ban MetaおよびMeta Wearables Device Access Toolkitの提供内容に依存する。
- 画像認識、音声認識、LLM処理には外部APIまたはローカル推論環境を利用する可能性がある。
- 実運用では、第三者の顔、会話、個人情報を扱う可能性があるため、プライバシー設計を必須とする。
- 長時間の常時キャプチャは、バッテリー、発熱、ストレージ、APIコストに影響する。
- Ray-Ban Metaにはディスプレイがないため、AR表示やHUD表示はMVP対象外とする。
- Meta AIの機能や音声コマンド連携は利用できない前提とする。
