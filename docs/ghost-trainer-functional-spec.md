# スマートグラス ゴースト・トレーナー 要求仕様書

## 1. 基本情報

| 項目 | 内容 |
| --- | --- |
| 文書名 | スマートグラス ゴースト・トレーナー 要求仕様書 |
| 対象アプリ | ボードゲーム対局支援アプリ「ゴースト・トレーナー」 |
| 初期対象ゲーム | オセロ |
| 対象デバイス | Ray-Ban Meta glasses + Android端末、またはスマホカメラ |
| 開発者 | 小松琢磨、中村壮馬 |
| 初期開発方針 | 実物オセロ盤面をカメラで読み取り、Egaroucidの解析結果をLLMで説明し、対局中支援をスマホ画面または音声で提示する |

## 2. 目的

本アプリは、実物のボードゲーム盤面をカメラで読み取り、既存ゲームAIとLLMを組み合わせることで、対局中の次の一手、候補手、読み筋、局面の考え方を提示するエンタメ型トレーニングアプリである。

初期MVPではオセロに対象を絞り、オセロAIにはEgaroucidを利用する。Egaroucidは合法手、候補手、評価値、読み筋を出す役割を担い、LLMはその結果を自然言語で説明する役割を担う。LLM単独で最善手や合法手を決定しない。

## 3. スコープ

### 3.1 MVPで実装する範囲

- Ray-Ban Metaまたはスマホカメラで実物オセロ盤面を撮影する。
- 撮影画像から8x8マス、黒石、白石、空きマスを推定する。
- 推定された盤面と手番をユーザーが手動補正できる。
- 現在局面から合法手を判定する。
- Egaroucidへ現在局面を渡し、候補手、評価値、読み筋を取得する。
- LLMがEgaroucidの解析結果をもとに、候補手の理由や注意点を短く説明する。
- 対局中支援として、次の一手、複数候補、短い音声助言、スマホ画面での詳細説明を提示する。
- 着手履歴、盤面画像、AI解析結果、LLM解説を保存し、対局後に復習できる。

### 3.2 MVPで実装しない範囲

- オセロ以外のゲーム対応。
- 完全自動で誤認識なく盤面を読み続ける常時認識。
- Ray-Ban Meta上のAR重畳表示やHUD表示。
- 競技・大会など、外部ルールで支援が禁止される場面での利用支援。
- LLM単独による候補手生成や勝敗評価。
- 対戦相手の同意を前提としない録画・録音・公開共有。

## 4. 想定ユーザー

| ユーザー | 説明 |
| --- | --- |
| プレイヤー | オセロを指しながら、対局中の助言や対局後の復習を行うユーザー |
| 観戦者 | 盤面解析や読み筋を見ながら対局を楽しむユーザー |
| 開発者 | 盤面認識、Egaroucid連携、LLM解説、Android UI、音声応答を実装する開発者 |

## 5. ユースケース

### UC-GT-000 対局中支援を開始する

| 項目 | 内容 |
| --- | --- |
| アクター | プレイヤー |
| 前提 | Androidアプリが起動しており、カメラ利用とAI連携が許可されている |
| トリガー | プレイヤーがゴースト・トレーナーを起動し、オセロ対局を開始する |
| 基本フロー | 1. アプリが対象ゲームとしてオセロを選択する。2. プレイヤーがカメラ入力元をRay-Ban Metaまたはスマホカメラから選択する。3. アプリが新規セッションを作成する。4. アプリが対局中支援モードを開始する。 |
| 完了条件 | オセロ対局支援セッションが開始され、盤面撮影を実行できる |

### UC-GT-001 実物盤面を読み取る

| 項目 | 内容 |
| --- | --- |
| アクター | プレイヤー |
| 前提 | 実物オセロ盤がカメラに映っている |
| トリガー | プレイヤーが盤面読み取りを実行する、またはアプリが支援タイミングでフレームを固定する |
| 基本フロー | 1. アプリが現在のカメラ画像を固定する。2. 画像処理が盤面領域と8x8マスを推定する。3. 各マスの黒石、白石、空きマスを推定する。4. アプリが手番を推定またはユーザー入力で取得する。5. アプリが盤面状態を表示する。 |
| 代替フロー | 認識信頼度が低い場合、ユーザーに補正画面を表示する |
| 完了条件 | 解析可能なオセロ局面が作成される |

### UC-GT-002 対局中に次の一手を確認する

| 項目 | 内容 |
| --- | --- |
| アクター | プレイヤー |
| 前提 | 盤面状態と手番が確定している |
| トリガー | プレイヤーが「次の一手」または同等の操作を行う |
| 基本フロー | 1. アプリが現在局面の合法手を判定する。2. アプリが局面をEgaroucidへ送信する。3. Egaroucidが候補手、評価値、読み筋を返す。4. LLMが解析結果を短い助言に変換する。5. アプリが最有力手を音声で提示し、スマホ画面に候補手一覧と詳細を表示する。 |
| 代替フロー | Egaroucid連携に失敗した場合、アプリは合法手一覧とエラー内容を表示する |
| 完了条件 | プレイヤーが次の一手と理由を確認できる |

### UC-GT-003 盤面認識結果を補正する

| 項目 | 内容 |
| --- | --- |
| アクター | プレイヤー |
| 前提 | 盤面認識結果に誤りがある |
| トリガー | プレイヤーが補正画面を開く、またはアプリが低信頼度を検出する |
| 基本フロー | 1. アプリが8x8盤面をスマホ画面に表示する。2. プレイヤーが誤ったマスをタップする。3. マス状態を黒、白、空きの順に切り替える。4. プレイヤーが手番を確認または変更する。5. アプリが補正後の局面を保存する。 |
| 完了条件 | 補正後の盤面で合法手判定とAI解析を実行できる |

### UC-GT-004 対局後に復習する

| 項目 | 内容 |
| --- | --- |
| アクター | プレイヤー |
| 前提 | 着手履歴と解析結果が保存されている |
| トリガー | プレイヤーが対局履歴を開く |
| 基本フロー | 1. アプリが対局履歴を一覧表示する。2. プレイヤーが復習したい対局を選択する。3. アプリが各局面の実着手、Egaroucid候補手、評価値、LLM解説を表示する。4. プレイヤーが悪手や分岐を確認する。 |
| 完了条件 | プレイヤーが対局中の判断を振り返れる |

## 6. 機能仕様

### 6.1 盤面入力・認識

| ID | 要件 |
| --- | --- |
| FS-GT-CAM-001 | アプリはRay-Ban Metaまたはスマホカメラからオセロ盤面画像を取得できること |
| FS-GT-CAM-002 | アプリは支援実行時の最新フレームを解析対象として固定できること |
| FS-GT-CAM-003 | アプリは画像から盤面領域と8x8マスを推定できること |
| FS-GT-CAM-004 | アプリは各マスの状態を黒石、白石、空きマスとして推定できること |
| FS-GT-CAM-005 | アプリは盤面認識結果に信頼度を付与できること |
| FS-GT-CAM-006 | アプリは認識失敗時に手動補正画面へ誘導できること |

### 6.2 オセロ局面管理

| ID | 要件 |
| --- | --- |
| FS-GT-STATE-001 | アプリはオセロの8x8盤面を内部状態として保持できること |
| FS-GT-STATE-002 | アプリは黒番、白番、終局状態を管理できること |
| FS-GT-STATE-003 | アプリは現在局面の合法手を判定できること |
| FS-GT-STATE-004 | アプリは違法手をAI助言として表示しないこと |
| FS-GT-STATE-005 | アプリはユーザーの実着手を履歴として保存できること |
| FS-GT-STATE-006 | アプリは盤面認識、手動補正、実着手を同一セッション内で紐付けられること |

### 6.3 Egaroucid連携

| ID | 要件 |
| --- | --- |
| FS-GT-AI-001 | アプリまたは自作AIサーバーは現在局面をEgaroucidへ渡せる形式に変換できること |
| FS-GT-AI-002 | Egaroucidから候補手、評価値、読み筋、解析深度を取得できること |
| FS-GT-AI-003 | 解析に時間がかかる場合、アプリは処理中状態を表示できること |
| FS-GT-AI-004 | Egaroucidが利用できない場合、アプリはエラーを表示し、局面保存または再試行を選べること |
| FS-GT-AI-005 | Egaroucidの解析結果はLLM入力用に構造化して保持できること |

### 6.4 LLM解説

| ID | 要件 |
| --- | --- |
| FS-GT-LLM-001 | LLMは盤面状態、候補手、評価値、読み筋を入力として解説を生成できること |
| FS-GT-LLM-002 | LLMはEgaroucidの解析結果に基づいて説明し、根拠のない最善手を生成しないこと |
| FS-GT-LLM-003 | LLMは対局中向けに短い音声助言を生成できること |
| FS-GT-LLM-004 | LLMは復習向けに詳細な局面解説、悪手理由、代替案を生成できること |
| FS-GT-LLM-005 | LLM解説には、解析結果の信頼度や盤面認識の不確実性を反映できること |

### 6.5 UI・音声提示

| ID | 要件 |
| --- | --- |
| FS-GT-UI-001 | アプリはスマホ画面に現在盤面、候補手、評価値、読み筋を表示できること |
| FS-GT-UI-002 | アプリは対局中に短い音声助言を再生できること |
| FS-GT-UI-003 | アプリは候補手を1手だけ表示する簡易モードと、複数候補を表示する詳細モードを切り替えられること |
| FS-GT-UI-004 | アプリは盤面認識結果を8x8グリッドとして表示し、ユーザーがマス状態を補正できること |
| FS-GT-UI-005 | アプリは対局中支援、練習、対局後復習のモードを切り替えられること |
| FS-GT-UI-006 | アプリは競技・大会など支援禁止場面では利用しない方針を明示できること |

### 6.6 履歴・復習

| ID | 要件 |
| --- | --- |
| FS-GT-LOG-001 | アプリは対局セッションを作成し、開始時刻、対象ゲーム、入力元を保存できること |
| FS-GT-LOG-002 | アプリは局面ごとに盤面画像、盤面状態、手番、認識信頼度を保存できること |
| FS-GT-LOG-003 | アプリはEgaroucid解析結果とLLM解説を局面に紐付けて保存できること |

## 7. データモデル

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | string | yes | 対局支援セッションID |
| game_type | string | yes | 初期値はothello |
| mode | string | yes | in_game、practice、review |
| input_source | string | yes | rayban_meta、phone_camera、manual |
| started_at | datetime | yes | セッション開始時刻 |
| ended_at | datetime | no | セッション終了時刻 |

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | string | yes | 局面ID |
| session_id | string | yes | 紐付くセッションID |
| board | string | yes | 64マスの状態。B/W/empty等で表現する |
| side_to_move | string | yes | blackまたはwhite |
| source_image_uri | string | no | 元画像の保存先 |
| recognition_confidence | number | no | 盤面認識信頼度 |
| corrected_by_user | boolean | yes | ユーザー補正の有無 |

### 7.3 OthelloAnalysis

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | string | yes | 解析ID |
| board_state_id | string | yes | 対象局面ID |
| engine | string | yes | 初期値はEgaroucid |
| best_move | string | no | 最有力手 |
| candidate_moves | array | yes | 候補手、評価値、読み筋の配列 |
| search_depth | number | no | 解析深度 |
| raw_result | object | no | エンジンの元出力 |

### 7.4 TrainerExplanation

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | string | yes | 解説ID |
| analysis_id | string | yes | 対象解析ID |
| short_advice | string | yes | 対局中に音声提示する短い助言 |
| detailed_explanation | string | no | スマホ画面や復習で表示する詳細解説 |
| warnings | array | no | 危険手、盤面認識不確実性、利用注意など |

## 9. 非機能要件

| ID | 分類 | 要件 |
| --- | --- | --- |
| NFS-GT-001 | 応答性 | 対局中支援では、盤面確定から助言提示までの遅延を短く抑えること |
| NFS-GT-002 | 正確性 | 合法手判定とEgaroucid解析結果を優先し、LLMの幻覚による違法手提示を防ぐこと |
| NFS-GT-003 | 補正性 | 盤面認識が誤った場合でも、ユーザーが短時間で修正できること |
| NFS-GT-004 | 説明性 | 音声助言は短く、詳細説明はスマホ画面または復習画面で確認できること |
| NFS-GT-005 | 拡張性 | 将来的に他のボードゲームを追加できるよう、ゲーム固有処理と共通UIを分離すること |
| NFS-GT-006 | 倫理・利用制限 | 競技・大会など支援が禁止される場面では利用しない方針を明示すること |

## 12. 実装順序

1. スマホカメラで実物オセロ盤面を撮影し、静止画として固定する。
2. 8x8盤面の手動入力UIを実装し、先に合法手判定を動かす。
3. 盤面画像からマス位置と石の色を推定する認識処理を追加する。
4. 盤面認識結果の手動補正UIを実装する。
5. Egaroucid連携を実装し、候補手、評価値、読み筋を取得する。
6. Egaroucid解析結果をLLMへ渡し、短い助言と詳細解説を生成する。
7. 対局中支援UIと音声提示を実装する。
8. 着手履歴、解析結果、LLM解説を保存し、復習画面を実装する。
9. Ray-Ban Meta実機で盤面撮影、遅延、音声提示、操作性を検証する。
