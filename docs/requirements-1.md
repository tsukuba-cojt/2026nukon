# nukon 要件定義書 v1

## 概要

既存のトイカメラはネットワークに繋がらず、撮影後の取り回しが煩雑である。デジカメは高いので買えない。
nukon は「撮ることそれ自体を楽しめる」専用機としての満足感と、「撮ったらすぐ手元の他のデバイスで扱える」現代的な利便性を両立させたカメラを作ることを目的とする。
あと、HWからSWまで一貫して作るのはおもろいのでやりたい。

### システムの位置づけ
- 単独で動作する撮影専用デバイス
- 撮影した画像は、ネットワーク経由でユーザーが普段使うデバイス（PC、スマートフォン等）に取り出せる
- SNSに投稿するなどのスマホでの便利なアプリケーション

### スコープ
**対象**
- 静止画の撮影
- 撮影に関わる物理操作（構図決定、ピント合わせ、シャッター）
- 撮影した画像のネットワーク経由での取り出し
- 撮影体験を支える物理筐体（本体・操作機）
- SNS への直接投稿


## 機能要件

利用者が nukon でできること：

1. **撮影する**
   - 電源を入れる／切る
   - ファインダーまたは画面を見ながら構図を決める
   - シャッターを押して写真を記録する
2. **その場で確認する**
   - 撮影直後の画像を画面で確認する
   - 過去に撮った画像を見返す
3. **設定を変える**
   - 撮影に関わる設定（モード等）を画面上で変更する
4. **離れた位置から撮る**
   - 本体から手を離した状態でも、別の操作機からシャッターを切れる
   - 本体に触れずに基本的な撮影操作ができる
5. **写真を取り出す**
   - 撮った画像をネットワーク経由で手元のデバイスに転送する

### 操作系の体験要件
- シャッターは「半押し（ピント合わせ）」「全押し（撮影）」の二段操作ができる
- 主たる操作は物理ボタンで完結し、画面タッチは補助とする
- ファインダーを覗きながらの撮影と、画面を見ながらの撮影の両方ができる

## 非機能要件

撮影体験を損なわないために以下を満たす：

- **応答性**：シャッターを押してから記録されるまでの遅延が、撮影体験を阻害しない程度であること
- **携帯性**：日常的に持ち歩ける重量・サイズに収まること
- **稼働時間**：1 回の外出（数時間程度）を、充電なしで使える電池容量を持つこと
- **接続性**：一般的な家庭・モバイル環境のネットワークに接続できること

> 具体的な数値（重量上限、駆動時間、シャッタータイムラグ等）は要求仕様の段階で確定する。

## 制限

### 動作環境
- 本機は **Raspberry Pi を中核** とする構成で実装する（プロジェクトとしての前提）
- 撮影モジュールは Raspberry Pi で扱えるカメラモジュールから選定する

### 担当
- 小松琢磨
- 中村相馬

## 付録：現時点での構成案

要件ではなく実装上の検討メモ。要件側が固まり次第、要求仕様書に正式に移す。

トイカメラは2つのモジュールからなる構成を想定している。

### メインモジュール

Raspberry Pi と大型のバッテリー、タッチパネルと EVF を備えたメインのカメラモジュール。

- Raspberry Pi 5
- 64MP カメラモジュール
  - https://www.arducam.com/64mp-af-for-raspberry-pi.html
  - AF / プログラマブルなフォーカスが要件
- EVF モジュール（Aliexpress に売っているもの）
  - アクセサリーシューに付ける形でも、本体に組み込んでも可
- タッチパネル
- バッテリーモジュール（自作？）
  - 電源ボタン・充電まわりが鬼門

### サブモジュール

シャッターボタンと矢印キーを備えた小型の操作モジュール。本体から離れた位置で撮影するために用いる。

- 2 段押しのできるシャッターボタン
  - ピエゾ素子＋ハプティックフィードバックで実現するか、物理 2 段スイッチで実現するかは要検討
- 矢印キー
- バッテリーモジュール（自作？）
  - 電源ボタン・充電まわりが鬼門
